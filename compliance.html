🛡 5-Gate Safety Architecture

Every prescription passes through 5 independent safety gates. Any single gate can block. Conservative failure by design — every error handler defaults to the most restrictive safe state.

1

Eligibility Engine (Pre-Consultation)

Hard stops: MTC/MEN2 history, SCOFF eating disorder screen (≥2 = blocked), PHQ-9 Q9 suicidal ideation (crisis block with Lifeline/000/Beyond Blue), BMI thresholds (ethnicity-adjusted for Asian, South Asian, Middle Eastern, Aboriginal/Torres Strait Islander), age 18–75, Australian state S4 regulations.

2

3-Tier SafetyValidator (9 Parallel Checks)

Tier 1 (<50ms, deterministic): 3,778 Supabase records — allergy cross-reactions, drug interactions, contraindications, dosing, TGA pregnancy, breastfeeding, food & supplement interactions, QT prolongation risk.
Tier 2 (~2s, Haiku 4.5 AI): Handles novel drugs, complex multi-drug interactions.
Tier 3 (<10ms, fail-safe): Conservative fallback — "Insufficient data, consult specialist."

3

ContentSafetyValidator (Real-Time AI Response)

Layer 1: Scans every 50 tokens during streaming for red flag symptoms without emergency direction, contraindicated recommendations.
Layer 2: Post-stream deep validation (5 dimensions) — medication safety, diagnosis completeness, urgency appropriateness. Haiku 4.5 generates corrected response if critical issues found.

4

OutputFilteringService (AHPRA/TGA Language Compliance)

Blocks forbidden diagnostic language ("I diagnose you with"), definitive claims ("you have diabetes"), prescription claims ("I am prescribing"), treatment decisions ("you must start treatment"). Enforces doctor-deferral language across all AI responses.

5

7-Agent Byzantine Consensus (Multi-Provider)

7 AI agents from 4 different providers (Claude Opus 4.6, Gemini 3 Flash, GPT-5.2, Perplexity Sonar Pro) must agree before any prescription approval. ANY 1 agent can VETO. Temperature locked at 0.2 — "safety cannot be creative." Eliminates single-model bias through provider diversity. Agents: Drug Interaction Validator, Allergy Checker, Contraindication Enforcer, Dose Validator, Pregnancy/Breastfeeding Officer, TGA Compliance Auditor, Safety Consensus Lead.

⚠ Cross-Border Data Flows (Transparency Disclosure)

The following services process data outside Australia. Mitigation: PHI anonymization before AI calls, Data Processing Agreements, patient consent to international transfer.

  • Anthropic Claude API (US) — conversation content with PHI anonymizer applied; direct identifiers stripped
  • Terra Wearables API (US) — body metrics (weight, steps, sleep)
  • Calendly (US) — appointment data (name, email)
  • Stripe (US) — payment card data (PCI DSS Level 1 compliant)
  • ElevenLabs (US) — synthesized voice audio

Mitigation: Anthropic offers data residency controls (Opus 4.6 feature) for AU region. Claude AI Anonymizer strips name, email, phone before calls. MediRecords (primary EHR) remains fully within Australian jurisdiction.
