How does Dr. Sophia handle data sovereignty and residency?
Data
▼

All infrastructure runs on AWS ap-southeast-2 (Sydney). Patient data never leaves Australian soil. We use AES-256 encryption at rest and TLS 1.3 in transit. Our EKS cluster runs in a dedicated VPC with private subnets. Database (Supabase) is hosted in Australia with row-level security policies.

If pressed on Anthropic/Claude API: "API calls to Claude go to Anthropic's API, but we send only de-identified clinical context - never raw PII. We're evaluating Anthropic's data residency options and AWS Bedrock (which keeps everything in-region) as we scale into government."

SaMD Class 1 Exempt under TGA guidelines - lowest risk category. We provide clinical decision support and advisory information, not autonomous diagnosis. A qualified clinician is always in the loop.

How do you ensure AI safety in clinical decisions?
Safety
▼

Multiple layers:

  • 42-Agent Consensus: Critical decisions go through Byzantine fault-tolerant multi-agent analysis. Multiple AI agents independently assess, consensus required
  • Safety Middleware: Every AI response passes through content safety validation before reaching clinician/patient
  • GLP-1 Safety Validator: Dedicated checks for medication recommendations (pregnancy, MTC/MEN2)
  • Crisis Protocol: Auto-detection of mental health crises with 5 Australian emergency hotlines
  • Clinical Decision Audit: Every recommendation logged - what went in, what came out, why
  • Human-in-the-loop: Clinician approval required for all treatment decisions

What encryption and data protection measures are in place?
Data
▼
  • At rest: AES-256. AWS EBS volumes encrypted. Supabase DB encryption
  • In transit: TLS 1.3. ALB terminates SSL with ACM-managed certificates
  • Application layer: Custom encryption module - context-aware, different keys per data type
  • Access control: Row-level security (RLS), RBAC, API key auth
  • Secrets: AWS Secrets Manager. Zero hardcoded secrets
  • Audit: All access/modifications logged with timestamps, user identity, action

Business
▼

Three value levers for public health:

For Adam: "HIVE saves 60,000 bed hours through inpatient remote monitoring. Dr. Sophia could deliver similar savings on the outpatient side - AI triaging, pre-consultation data gathering, and structured follow-ups that reduce unnecessary ED presentations."
